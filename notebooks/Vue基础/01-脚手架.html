    # SPA简介

    单页面应用程序简称SPA

    在一个web网站中只有一个html文件

    它们基于唯一的页面来完成

    # vue-cli搭建

    1. 全局安装

      npm install -g @vue/cli
//安装vue-cli 4

    2. 搭建脚手架

      vue create 项目名

    3. 运行

      npm run serve

    # vue项目运行流程

    通过main.js把APP.vue渲染到index.html中的指定区域内

    main.js

    //导入vue包，得到Vue构造函数
import Vue from 'vue'
//导入APP.vue根组件，把其中的模板结构渲染到页面中
import App from './App.vue'

//创建Vue的实例对象
new Vue({
    //把render函数指定的组件渲染到页面上
  render: h => h(App),
}).$mount('#app')

new Vue({
  el: "#app"
    //把render函数指定的组件渲染到页面上
  render: h => h(App),
})

    注意：$mount与el的挂载效果是一样的

    # 使用组件的三个步骤

    1. 使用import导入我们需要的组件

      import Left from '@/components/Left.vue'

    2. 使用components节点注册组件

      export default {
    components:{

    }
}

    3. 使用组件

      <Left></Left>

    # 全局和私有组件的使用

    1. 私有组件

      只需要将组件导入我们需要的页面内，它就是这个页面的私有组件，其他不可用

    2. 全局组件

      我们需要在main.js中来配置全局组件

      //main.js
import Left from '@/components/Left.vue'

参数1：注册名称
参数2：全局注册组件名
Vue.component('MyCount',Left)

    # 组件的props

    使用props（组件的自定义属性）合理利用props可以提高组件的复用性

    1. 使用

      export default {
    props:['init','自定义属性'],
    data() {
        return {
            
        }
    }
}

      注意：

      <!--注意这里我们使用属性的形式那么传值过来的是字符串-->
<MyCount init="9"></MyCount>
<!--使用bind形式则传来的为数字格式-->
<MyCount :init="9"></MyCount>

    2. 默认props

      当我们没有传值时可以设定默认值,这个时候props为对象格式

      props:{
        init:{
            default:0
        }
    },

    3. props的type值类型

      props:{
        init:{
            default:0,
            type:Number
        }
    },

    4. props的require必填项

      使用组件时必须使用这个参数否则会报错

      props:{
        init:{
            default:7,
            type:Number,
            require:true
        }
    },

    当然不可能将所有的节点都添加属性

    # 组件的生命周期&生命周期函数

    1. 生命周期

      生命周期是指组件从创建->运行->销毁，整个阶段，强调的是一个时间段

    2. 生命周期函数

      是由vue提供的内置函数，会伴随着生命周期，自动执行

      1. 创建阶段：
        • beforeCreate：在组件创建之前——组件的props&data&methods还没有被创建，都处于不可用的状态。
        • created：创建好执行（为渲染只存在内存中）——组件的props&data&methods都已经创建好，处于可用状态，但是组件的模板结构尚未生成，这个地方我们经常会发起ajax请求拿到页面数据
        • beforeMount：组件将要渲染时执行——将要把内存中编译好的HTML结构渲染到浏览器中，此时浏览器中还没有当前组件的dom结构
        • mounted：已经渲染——dom结构已经渲染，即我们可以在这里操作DOM
      2. 运行阶段：
        • beforeUpdate——当数据改变时触发该函数（页面还没有渲染改变的数据）
        • updated运行后——根据最新的数据完成了对组件的重新渲染
      3. 销毁阶段
        • beforeDestroy
        • destroyed销毁后

    3. 兄弟组件之间的数据共享

      在vue2.x中兄弟之间数据共享的方案是EventBus

      //兄弟组件A（数据发送方）
import bus from './eventBus.js'
export default {
    data() {
        return {
            msg:''
        }
    },
    methods:{
        sendMsg(){
            bus.$emit('share',this.msg)
            //发送数据
        }
    }
}

      //eventBus.js
import Vue from 'vue'

//向外共享Vue的实例对象
export default new Vue()

      //兄弟组件C（数据接收方）
import bus from './eventBus.js'

export default {
    data(){
        return {
            msgFromLeft:''
        }
    }，
    created(){
        bus.$on('share',val=>{
            this.msgFromLeft = val
        })
        //数据接受，当share被触发时执行这个函数
    }
}

    # ref的引用

    在不依赖jq的时候获取dom或者组件的引用

    在每一个vue实例上，都包含一个$refs对象，里面存储着对应的DOM元素或者组件的引用

    在默认情况下$refs指向一个空对象

    <h1 ref="myh">ref</h1>

    console.log(this.$refs.myh)

    this.$nextTick(cb)

    将该回调函数内的代码延迟执行即它会把cb回调推迟到下一个DOM更新周期后执行

    this.$nextTick(() => {
    this.$refs.iptRef.focus()
})
